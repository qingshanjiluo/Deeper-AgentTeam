# 消息测试插件

用于诊断DeepSeek AgentTeam扩展中消息发送显示"undefined"问题的测试工具。

## 功能特点

1. **智能输入框检测** - 支持多种DeepSeek输入框选择器
2. **多种发送方式** - 按钮点击、键盘事件、表单提交、自定义事件
3. **详细日志记录** - 记录所有操作和结果
4. **特殊测试用例** - 测试空消息、undefined、null等边界情况
5. **实时状态监控** - 显示页面连接状态和诊断信息

## 安装方法

1. 打开Chrome浏览器，访问 `chrome://extensions/`
2. 启用右上角的"开发者模式"
3. 点击"加载已解压的扩展程序"
4. 选择 `deeper-AgentTeam/test-message-plugin` 目录

## 使用方法

### 基本测试
1. 打开DeepSeek聊天页面（https://chat.deepseek.com）
2. 点击扩展图标打开测试插件
3. 点击"刷新状态"检查页面连接
4. 输入测试消息，点击"发送测试消息"

### 特殊测试
- **空消息测试**：测试空字符串处理
- **undefined测试**：测试undefined值处理
- **null测试**：测试null值处理
- **数字测试**：测试非字符串值

### 测试类型
- **直接发送**：发送到当前标签页
- **模拟发送**：不实际发送，仅测试消息处理
- **创建新标签页**：打开新DeepSeek页面并发送

## 诊断"undefined"问题

### 可能原因
1. **消息传递问题**：后台到内容脚本的消息丢失
2. **类型转换错误**：非字符串值未正确转换
3. **异步处理问题**：消息在异步操作中变为undefined
4. **API限制**：Chrome扩展API的某些限制

### 诊断步骤
1. 使用测试插件发送各种类型的消息
2. 观察测试结果中的"原始消息"和"处理后的消息"
3. 检查是否有错误信息
4. 查看详细日志了解发送过程

## 技术实现

### 输入框查找策略
1. DeepSeek特定选择器（data-testid、类名等）
2. 通用选择器（textarea、contenteditable等）
3. 占位符匹配
4. 位置检测（页面底部）

### 发送策略
1. **按钮点击**：查找并点击发送按钮
2. **键盘事件**：模拟Enter键和Ctrl+Enter
3. **表单提交**：尝试提交父表单
4. **自定义事件**：触发可能的自定义发送事件

### 消息验证
1. 检查消息类型和内容
2. 验证输入框是否成功接收文本
3. 检测发送后输入框是否被清空
4. 记录所有操作结果

## 常见问题

### Q: 消息输入正常但没有发送出去
A: 可能是发送按钮查找失败。尝试：
1. 使用"键盘发送"备选方案
2. 检查控制台日志了解按钮查找详情
3. 手动点击页面上的发送按钮

### Q: 测试结果显示"undefined"
A: 检查：
1. 原始消息类型是否正确
2. 消息处理逻辑是否有类型转换
3. 内容脚本是否正确接收消息

### Q: 页面状态显示"未找到输入框"
A: 可能原因：
1. 页面不是DeepSeek聊天页面
2. 页面结构已更改
3. 内容脚本未正确加载

## 调试技巧

1. **查看控制台日志**：F12打开开发者工具，查看Console标签
2. **检查网络请求**：查看消息发送后的网络请求
3. **元素检查**：检查输入框和发送按钮的DOM结构
4. **重新加载扩展**：在扩展管理页面重新加载测试插件
