#深度代理团队
[![许可证：MIT](https://img.shields.io/badge/License-MIT-yellow.svg)](https://opensource.org/licenses/MIT)
DeepSeek(https://chat.deepseek.com)设计的 Chrome 扩展，用于创建和管理多角色 AI 团队。它实现了
---
## ✨ 功能特性
- **团队管理**：创建团队，设置通用提示词、辅助提示词（定期强化角色）和团队介绍。
- **成员管理**：添加成员，为每个成员指定名称、身份标识、上级关系、专属提示词等。
- **自动初始化**：创建团队后，自动为每个成员打开独立的 DeepSeek 标签页，并发送初始化消息，让 AI 知晓自己的角色和团队背景。
- **串行消息传递**：从起始成员开始，按上级关系逐级向下传递消息，每个成员的回复自动追加到上下文，确保信息完整。
- **实时聊天面板**：在团队面板中实时显示所有成员的对话，支持折叠/展开，并可向特定成员私聊。
- **辅助提示词插入**：根据配置的轮数阈值，自动在合适时机向成员插入辅助提示词，防止角色遗忘。
- **JSON 指令解析**：成员回复中可包含 JSON 指令（如 `{"type":"send","target":"成员","message":"..."}`），实现成员间自主协作。
- **本地存储**：所有团队数据、对话历史均存储在本地，不收集任何个人信息。

## 📦 安装
### 从源码加载（开发者模式）
1. 下载本仓库源码到本地：
   ```bash
   git 克隆 https://github.com/yourusername/deeper-agentteam.git
   ```
2. 打开 Chrome 浏览器，进入扩展管理页面 `chrome://extensions/`。
3. 开启右上角的“开发者模式”。
4. 点击“加载已解压的扩展程序”，选择项目文件夹 `deeper-agentteam`。
5. 插件图标将出现在工具栏中，点击即可打开团队面板。
> **注意**：插件仅在 [https://chat.deepseek.com](https://chat.deepseek.com) 下生效。

## 🚀 快速开始
### 1. 创建团队
点击工具栏图标打开团队面板，点击“+ 新建团队”，填写以下信息：
- **团队名称**：给你的团队起个名字。
- **通用提示词**：所有成员共享的上下文，例如团队目标、协作原则等。
- **辅助提示词**：用于定期强化成员角色的文本，例如“你是一名资深产品经理，请保持专业”。
- **团队介绍**：简短的团队描述。
### 2. 添加成员
在团队详情页点击“+ 添加成员”，填写：
- **成员名称**：例如“产品经理”、“前端开发”。
- **身份标识**：唯一标识符，如 `pm_001`。
- **所属上级**：该成员向谁汇报（用于串行传递）。顶级成员可不填。
- **专属提示词**：描述该成员的具体职责、性格、技术栈等。
- **工作流程**（可选）：简要描述该成员的工作步骤。
- **成员简介**（可选）：补充说明。
点击“添加”保存成员。
### 3. 初始化成员标签页
创建团队并添加成员后，插件会自动为每个成员打开一个独立的 DeepSeek 标签页（后台打开，不激活），并发送初始化消息，将团队信息、角色提示词等一次性发送给 AI。初始化完成后，成员即可开始工作。
> 您可以手动打开这些标签页查看成员状态，但插件会自动管理，无需干预。
### 4. 发起串行群发
在团队面板的聊天区域：
1. 选择起始成员（通常为顶级成员，如“产品经理”）。
2. 输入消息内容，例如：“我们需要开发一个用户登录功能，请开始处理。”
3. 点击“串行发送”按钮。
插件将自动执行以下操作：
- 向起始成员发送消息（附带当前上下文）。
- 等待成员回复，将回复追加到上下文。
- 查找该成员的下级，按顺序向每个下级发送更新后的上下文，并等待回复。
- 重复此过程，直到所有相关成员处理完毕。
最终结果将显示在面板中，并附有完整的讨论链。
### 5. 私聊与面板同步
- 在面板中，您可以选择某个成员，输入消息后点击“发送”，即可向该成员私聊（不触发串行流程）。
- 所有成员的回复都会实时同步到面板的聊天区域，按成员分组显示。
### 6. 辅助提示词插入
插件会为每个成员维护一个对话轮数计数器。当达到预设阈值（默认5轮）时，下一次向该成员发送消息时，会自动在提示词前插入**辅助提示词**，以强化角色设定。您可以在团队设置中调整阈值。

## ⚙️ 配置说明
### 团队设置
- **通用提示词**：所有成员在每次对话中都会看到的提示词。
- **辅助提示词**：定期插入的强化文本，可包含角色描述、工作原则等。
- **辅助插入轮数**：每 N 轮对话插入一次辅助提示词，默认 5。
### 成员属性
| 字段 | 说明 | 示例 |
|------|------|------|
| 成员名称 | 显示名称，必须唯一 | 产品经理 |
| 身份标识 | 用于内部识别，建议用英文缩写 | pm_001 |
| 所属上级 | 上级成员名称（必须与已有成员一致） | 无 |
| 专属提示词 | 描述该成员的核心职责和特点 | 你擅长需求分析... |
| 工作流程 | 可选，描述工作步骤 | 需求收集→分析→验收 |
| 成员简介 | 可选，补充说明 | 有5年B端产品经验 |
